EXAMPLE-9 : Using Yield
	object hello {

	def main(args: Array[String]) = {
		val ints = List(1, 2, 3, 4, 5)
		val doubles = for { i <- ints } yield (i * 2)
		println(s"output is = $doubles")
		}
	}
The for keyword has even more power: When you use the yield keyword instead of do, you create for expressions which are used to calculate and yield results. This code creates a new list, where the value of each element in the new list is twice the value of the elements in the original list.

EXAMPLE-16 : Financial App - Bond Yield to maturity calculator
 import scala.annotation.tailrec
import scala.math.{pow, round}

object BondExample extends App {
  val bond = new Bond("BuBa 4Y 8,5%", 100, 6.5, 4)
  val YTM: Double = round(bond.yieldToMaturity(115, 0.0, 0) * 100.0) / 100.0
  println("\n----------------------------------------------------")
  println("The yield to maturity is approximately " + YTM + " %")
  println("----------------------------------------------------\n")
}

trait Formulae {
  def discount(r: Double, t: Long): Double = pow((1 + r / 100), -t)

  def cashFlowsPresentValue(r: Double, cf: Seq[Double]): Double = {
    val idx = cf.indices.map(_ + 1) // E.g. period numbers
    val values: Seq[Double] = (cf zip idx) map { case (c, t) =>
      c * discount(r, t)
    }
    values.foldLeft(0.0)(_ + _)
  }
}

case class Bond(name: String, principal: Double, coupon: Double, periods: Long) extends Formulae {
  val cashflows: Seq[Double] = Seq.fill(periods.toInt - 1)(coupon) ++ Seq(coupon + principal)
  def initialNPV(r: Double): Double = cashFlowsPresentValue(r, cashflows)

  @tailrec final def yieldToMaturity(price: Double, ytm: Double, precision: Long): Double = {
    println("Trying with ytm = " + ytm)
    val inpv: Double = initialNPV(ytm)
    if (price == inpv || precision > 5) {
      println("This value seems to be OK"); ytm
    }
    else {
      val nprec: Long = (price < inpv) == (precision % 2 == 0) match {
        case false => precision + 1
        case (_) => precision
      }
      val nytm: Double = (price < inpv) match {
        case true => round(pow(10, nprec) * (ytm + pow(10, -nprec))) / pow(10, nprec)
        case (_) => round(pow(10, nprec) * (ytm - pow(10, -nprec))) / pow(10, nprec)
      }
      yieldToMaturity(price, nytm, nprec)
    }
  }
} // end of case class Bond
Financial app that calculates a Bond's Yield to maturity. Showcases how we can use a class to calculate. Also showcases the use of App trait

EXAMPLE-19 : Extract value from JSON array 
import play.api.libs.json._

case class Car(Brand: String, Model: String, Type: String, Price: String)

object JsonParser {
  implicit val carFormat = Json.format[Car]

  def parseJson(jsonString: String): Seq[Car] = {
    val json = Json.parse(jsonString)
    val cars = json.as[Seq[Car]]
    cars
  }

  def main(args: Array[String]): Unit = {
    val jsonString =
      "[{\"Brand\":\"toyota\",\"Model\":\"corolla\",\"Type\":\"4d\",\"Price\":\"5000\"}," +
        "{\"Brand\":\"honda\",\"Model\":\"acord\",\"Type\":\"2d\",\"Price\":\"7200\"}]"
    val cars = parseJson(jsonString)
    println("BRAND" + "," + "MODEL" + "," + "TYPE" + "," + "PRICE")
    cars.foreach(car => println(s"${car.Brand}, ${car.Model}, ${car.Type}, ${car.Price}"))
  }
}
The above program extracts value from an array and prints the key values in a comma delimited form with each occurrence in a separate line. Also one needs to ensure that the play library is downloaded as dependent in the same way as scala.xml was downloaded as per this document.

Why Data Engineers should learn Scala ?


1. Scala stands for scalable language: one of the foundational ideas of developing Scala. It is meant to be big data-friendly, allow seamless integration with existing stacks, and grow in a "bazaar" fashion where developers can update its design with time. Moreover, everything in Scala is customizable to the user's programming requirements.

2. Due to its functional programming + OOP approach, all data structures are represented by case classes -- making customization extremely approachable.

3. Compile-time type inference adds to code brevity and comes immensely handy when dealing with complex pipelines. Moreover, data immutability (as we'll see below) and persistent data structures simplify running tests and help make the app more reliable.

4. Tools such as Apache Spark, Spark SQL, Databricks, and Kafka both use Scala under the hood; this has greatly increased Scala's large-scale credibility.

5. Combining concepts of conciseness and functional paradigm with OOP and high-level performance, data engineers can use Scala equally for lightweight, user-facing applications and terabyte-level big data pipelines with Spark jobs and distributed systems.

6. Niche tools like Quantexa (Links entities between third party data) have their integration pipelines built using Scala. Thus for a company that plans to use tools like Quantexa need Data Engineers with Scala expertise to integrate such tools with their Data Cloud Ecosystem.
